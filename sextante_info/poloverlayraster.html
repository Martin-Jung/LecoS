Overlay raster metrics (Polygons)

Brief Description

Overlays a raster with a given polygon vector layer and allows the user to compute a selected land cover metric per feature.
Ether landscape metrics or general statistics can be computed for every feature. Output results as table but can be added to the attribute table of the polygon vector layer as well.
Look here for an explanation of some of the landscape metrics

Parameters

  • [param] Input landscape raster: Classified raster to work on

  • [param] Input polygon layer: Polygon vector layer with multiple features

  • [param] Raster classified?: Should a landscape metric on a given class or normal statistics be computed?

  • [param] Landscape class: The value of the pixels to compute on

  • [param] Landscape metric: The metric to be calculated. Currently supported: Total Land cover, Landscape Proportion, Edge length, Edge density, Number of Patches, Patch density, Greatest patch area, Smallest patch area, Mean patch area, Median patch area, Mean patch distance, Mean patch shape ratio, Overall Core area, Landscape division, Effective Meshsize and the Splitting Index

  • [param] Normal statistics: Normal statistic to be computed instead. Currently supported: Mean, sum, minimum, maximum, standard deviation, lower quantile, median, upper quantile, shannon diversity index, eveness or simpson diversity index.

  • [param] Output Table: Output result as table (can be added to input polygon layer as well)

Limitations

  • The rasterlayer needs to have a valid no-data value. (for instance 0)
  • The cells in the rasterlayer need to be real squares
  • Only the first band of the raster layer is used
  • Very large raster layers might prevent this tool from running due to memory issues.
  • Both layers need to have the same spatial projection
  • Works only on polygon layers
  • Polygon vector layer has to overlay the landscape
  • Values can be added to the attribute table, but sometimes don't appear immediately.
